Gmail Session Automation

Complete example of capturing and restoring Gmail authentication

Playwright OAuth2 Session Management

This example demonstrates how to capture a fully authenticated Gmail session and restore it in a different browser instance, eliminating the need for repeated authentication.

1. Capture Session

import { Session } from '@psp/core';
import { PlaywrightAdapter } from '@psp/adapter-playwright';
import { chromium } from 'playwright';

async function captureGmailSession() {
  const browser = await chromium.launch({ 
    headless: false 
  });
  const page = await browser.newPage();
  const adapter = new PlaywrightAdapter(page);

  // Create session
  const session = await Session.create({
    name: 'Gmail Production',
    description: 'Authenticated Gmail session',
    tags: ['gmail', 'production'],
    storage: 'local'
  }, adapter);

  // Navigate and authenticate
  await page.goto('https://gmail.com');
  // User performs login manually...
  
  // Wait for authentication complete
  await page.waitForURL('**/mail/u/**');
  
  // Capture the authenticated session
  await session.capture();
  
  console.log(`Session captured: ${session.getId()}`);
  await browser.close();
}

2. Restore Session

async function restoreGmailSession(sessionId) {
  const browser = await chromium.launch({ 
    headless: false 
  });
  const page = await browser.newPage();
  const adapter = new PlaywrightAdapter(page);

  // Load existing session
  const session = await Session.load(sessionId);
  
  // Restore session state
  await session.restore(page);
  
  // Navigate to Gmail - already authenticated!
  await page.goto('https://gmail.com');
  
  // Session is restored, user is logged in
  await page.waitForSelector('[role="main"]');
  
  console.log('Gmail session restored successfully!');
}

// Usage
const sessionId = await captureGmailSession();
await restoreGmailSession(sessionId);

AWS Console Automation

Persist complex AWS Console sessions across automation scripts

Selenium MFA Role Switching

Handle complex AWS Console workflows including MFA authentication, role switching, and service-specific state management. 

import { Session } from '@psp/core';
import { SeleniumAdapter } from '@psp/adapter-selenium';
import { Builder } from 'selenium-webdriver';

async function captureAWSSession() {
  const driver = await new Builder().forBrowser('chrome').build();
  const adapter = new SeleniumAdapter(driver);

  const session = await Session.create({
    name: 'AWS Console Admin',
    description: 'Administrator access with MFA',
    tags: ['aws', 'admin', 'mfa'],
    storage: 'redis' // Use Redis for sensitive sessions
  }, adapter);

  // Navigate to AWS Console
  await driver.get('https://console.aws.amazon.com');
  
  // Handle MFA authentication...
  // User completes login with MFA
  
  // Switch to specific role
  await switchToRole(driver, 'ProductionAdmin');
  
  // Navigate to specific service (e.g., EC2)
  await driver.get('https://console.aws.amazon.com/ec2/');
  
  // Capture the complete session state
  await session.capture();
  
  console.log('AWS session captured with role and service state');
  await driver.quit();
}

MCP Server Setup

// Start PSP MCP Server
npx @psp/mcp-server

// Or programmatically
import { PSPMCPServer } from '@psp/mcp';

const server = new PSPMCPServer();
await server.start(3001);
