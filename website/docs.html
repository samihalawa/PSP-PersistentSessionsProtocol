Getting Started with PSP

PSP (Persistent Sessions Protocol) is a unified approach to browser session management across automation frameworks. This guide will help you get up and running with PSP in minutes.

1. Install

Quick installation via npm or direct download

2. Configure

Set up your preferred storage and adapters

3. Automate

Start capturing and restoring sessions

What makes PSP special?

Unlike other session management tools, PSP is framework-agnostic and works across Playwright, Selenium, Puppeteer, and cloud browser providers. One session can be captured in Playwright and restored in Selenium seamlessly.

Installation

NPM Installation (Recommended)

# Install PSP CLI globally
npm install -g @psp/cli

# Install core library for your project
npm install @psp/core

# Install framework-specific adapters
npm install @psp/adapter-playwright
npm install @psp/adapter-selenium

From Source

# Clone the repository
git clone https://github.com/samihalawa/PSP-PersistentSessionsProtocol.git
cd PSP-PersistentSessionsProtocol

# Install dependencies and build
npm install
npm run build

# Link CLI globally
npm link

Docker Installation

# Pull PSP server image
docker pull psp/server:latest

# Run PSP server
docker run -p 3000:3000 -v psp-data:/data psp/server:latest

System Requirements

  • Node.js: Version 16.0 or higher
  • Operating System: Windows, macOS, or Linux
  • Memory: Minimum 512MB RAM
  • Storage: 100MB free space
  • Browser: Chrome, Firefox, Safari, or Edge (for web UI)

Basic Usage Example

import { Session } from '@psp/core';
import { PlaywrightAdapter } from '@psp/adapter-playwright';
import { chromium } from 'playwright';

// Create a new session
const browser = await chromium.launch();
const page = await browser.newPage();
const adapter = new PlaywrightAdapter(page);

const session = await Session.create({
  name: 'Gmail Session',
  tags: ['gmail', 'production'],
  storage: 'local'
}, adapter);

// Navigate and authenticate
await page.goto('https://gmail.com');
// ... perform login ...

// Capture session after authentication
await session.capture();

await browser.close();

Restore Session Later

// Later, in a different script or framework
const browser = await chromium.launch();
const page = await browser.newPage();
const adapter = new PlaywrightAdapter(page);

// Load and restore the session
const session = await Session.load('session-id');
await session.restore(page);

// Navigate to Gmail - already authenticated!
await page.goto('https://gmail.com');

Core Concepts

Sessions

A session represents the complete state of a browser at a specific point in time. It includes cookies, localStorage, sessionStorage, authentication tokens, and optionally recorded user interactions.

Session Components:
  • • Metadata: Name, description, tags, timestamps
  • • Storage State: Cookies, localStorage, sessionStorage
  • • Authentication: OAuth tokens, JWT tokens, API keys
  • • Navigation: Current URL, history
  • • Recording: User interactions (optional)

Adapters

Adapters provide framework-specific implementations for capturing and restoring browser state. Each adapter knows how to interact with its specific automation framework.

Available Adapters:
  • • PlaywrightAdapter: For Microsoft Playwright
  • • SeleniumAdapter: For Selenium WebDriver
  • • PuppeteerAdapter: For Google Puppeteer
  • • BrowserbaseAdapter: For Browserbase cloud
  • • SkyveriAdapter: For AI automation

Storage Providers

Storage providers handle where and how sessions are persisted. PSP supports multiple storage backends for different use cases and scales.

Storage Options:
  • • Local: File system storage (default)
  • • Redis: In-memory cache with persistence
  • • Database: PostgreSQL, MySQL, MongoDB
  • • Cloud: AWS S3, Google Cloud Storage, Azure

MCP Integration

Model Context Protocol (MCP) integration allows AI agents and tools to interact with PSP through standardized interfaces. Perfect for AI-driven automation workflows.

MCP Tools (12 available):
  • • psp_list_sessions - List all available sessions
  • • psp_create_session - Create new session
  • • psp_capture_session - Capture browser state
  • • psp_restore_session - Restore session
  • • psp_delete_session - Delete session

Framework Adapters

Storage Providers

Choose the right storage backend for your PSP sessions based on your scale, performance, and durability requirements.

Local File Storage (Default)

Simple file-based storage for development and small-scale deployments.

Use Cases: Development, testing, single-machine deployments 
const session = await Session.create({
  storage: 'local',  // Default
  name: 'My Session'
});

Redis Storage

High-performance in-memory storage with optional persistence.

Use Cases: High-throughput applications, distributed systems, caching 
const session = await Session.create({
  storage: 'redis',
  storageOptions: {
    host: 'localhost',
    port: 6379,
    password: 'your-password'
  }
});

CLI Reference

Complete reference for the PSP command-line interface.

psp list

List all available sessions with filtering options.

psp list                     # List all sessions
psp list --captured          # Show captured browser sessions
psp list --tags gmail,prod   # Filter by tags
psp list --status active     # Filter by status

psp create

Create a new PSP session.

psp create "Session Name"               # Basic creation
psp create "Gmail" "Description" -t gmail,prod  # With tags
psp create --storage redis                     # Specify storage

psp launch

Launch interactive browser for session capture.

psp launch                              # Launch browser
psp launch --platform gmail               # Target specific platform
psp launch --url https://github.com        # Navigate to URL
psp launch --profile production           # Use specific profile

psp ui

Launch the PSP web interface.

psp ui                       # Start on default port 3000
psp ui --port 8080           # Custom port
psp ui --host 0.0.0.0        # Bind to all interfaces

Troubleshooting

Common issues and their solutions when working with PSP.

Build Errors

If you encounter TypeScript compilation errors:

npm run build:clean  # Clean build artifacts
npm install           # Reinstall dependencies
npm run build         # Rebuild project

Session Capture Issues

If session capture fails:

  • • Ensure browser has completed authentication
  • • Check that cookies are not expired
  • • Verify storage provider is accessible
  • • Try increasing capture timeout
